### Step 1.2：扩展 ThreadState 为共享黑板（1 天）

**目标文件**：`backend/src/agents/thread_state.py`

**改造内容**：在现有 `ThreadState` 基础上新增黑板字段：

```python
from typing import Literal

class TaskStatus(TypedDict):
    task_id: str
    description: str
    assigned_agent: str | None
    status: Literal["PENDING", "RUNNING", "DONE", "FAILED"]
    result: str | None
    hitl_required: bool
    hitl_approved: bool | None

class BlackboardState(TypedDict):
    task_pool: NotRequired[list[TaskStatus] | None]    # 平铺任务池（替代原 task_stack）
    verified_facts: NotRequired[dict[str, Any] | None] # 跨域已验证事实
    route_count: NotRequired[int]                       # 防死循环计数器（MAX=12）
    planner_goal: NotRequired[str | None]               # Planner 的整体目标

class ThreadState(AgentState, BlackboardState):  # 多继承扩展
    sandbox: NotRequired[SandboxState | None]
    thread_data: NotRequired[ThreadDataState | None]
    title: NotRequired[str | None]
    artifacts: Annotated[list[str], merge_artifacts]
    todos: NotRequired[list | None]
    uploaded_files: NotRequired[list[dict] | None]
    viewed_images: Annotated[dict[str, ViewedImageData], merge_viewed_images]
```

### Step 1.5：LangGraph 图重构（2 天）

**目标文件**：`backend/src/agents/__init__.py` 或新建 `backend/src/agents/graph.py`

将现有单节点图替换为多节点图：

```
[START]
  → planner_node        # 任务拆解，写入 task_pool
  → router_node         # 读 task_pool，分配 assigned_agent
  → parallel_executor   # asyncio.gather 并行执行域 Agent
  → planner_node        # 结果汇总，判断目标是否达成
  → [END]
```

用 LangGraph 的 `StateGraph` 构建：

```python
graph = StateGraph(ThreadState)
graph.add_node("planner", planner_node)
graph.add_node("router", router_node)
graph.add_node("executor", parallel_executor_node)
graph.add_conditional_edges("planner", route_after_planner)  # DONE → END, HAS_TASKS → router
graph.add_edge("router", "executor")
graph.add_edge("executor", "planner")   # 执行完毕回 Planner 校验
```

## 三、第二阶段：共享黑板 + 并行执行（约 7 天）

### Step 2.1：task_pool 状态设计与 Reducer（2 天）

**目标文件**：`backend/src/agents/thread_state.py`（继续扩展）

实现 `merge_task_pool` reducer，支持：
- 并行 Agent 同时写回结果时的无冲突合并（按 `task_id` 合并）
- 状态机转换保护（`RUNNING` 不可被外部重置为 `PENDING`）

```python
def merge_task_pool(
    existing: list[TaskStatus] | None,
    new: list[TaskStatus] | None
) -> list[TaskStatus]:
    """按 task_id 合并，new 中的状态更新覆盖 existing"""
    if not existing:
        return new or []
    if not new:
        return existing
    existing_map = {t["task_id"]: t for t in existing}
    for task in new:
        tid = task["task_id"]
        if tid in existing_map:
            existing_map[tid].update(task)  # 增量更新字段
        else:
            existing_map[tid] = task
    return list(existing_map.values())
```

### Step 2.2：并行调度器（3 天）

**新建文件**：`backend/src/agents/executor/parallel_executor.py`

```python
import asyncio

async def parallel_executor_node(state: ThreadState, config: RunnableConfig) -> dict:
    """并行执行分配给不同域 Agent 的子任务"""

    running_tasks = [t for t in state["task_pool"] if t["status"] == "PENDING"]

    # 按域 Agent 分组
    agent_tasks: dict[str, list[TaskStatus]] = {}
    for task in running_tasks:
        agent_name = task["assigned_agent"]
        agent_tasks.setdefault(agent_name, []).append(task)

    # asyncio.gather 并行执行
    coroutines = [
        _invoke_domain_agent(agent_name, tasks, config)
        for agent_name, tasks in agent_tasks.items()
    ]
    results = await asyncio.gather(*coroutines, return_exceptions=True)

    # 将结果写回 task_pool
    updated_tasks = _merge_results(running_tasks, results)
    return {"task_pool": updated_tasks, "route_count": state.get("route_count", 0) + 1}


async def _invoke_domain_agent(agent_name: str, tasks: list, config: RunnableConfig):
    """实例化域 Agent 并执行任务"""
    from src.agents.lead_agent.agent import make_lead_agent

    agent_config = RunnableConfig(configurable={
        **config.get("configurable", {}),
        "agent_name": agent_name,
    })
    agent = make_lead_agent(agent_config)  # 复用现有 make_lead_agent + YAML 配置

    task_description = "\n".join(t["description"] for t in tasks)
    result = await agent.ainvoke(
        {"messages": [HumanMessage(content=task_description)]},
        config=agent_config,
    )
    return result
```

## 四、第三阶段：HITL + 流程控制（约 5 天）

### Step 3.1：HITL 中间件（2 天）

**新建文件**：`backend/src/agents/middlewares/hitl_middleware.py`

利用 DeerFlow 现有的中间件 Hook 机制：

```python
class HITLMiddleware:
    """敏感操作拦截中间件，在 before_agent 阶段检测并触发人工审批"""

    async def before_agent(self, state: ThreadState, config: RunnableConfig):
        agent_name = config.get("configurable", {}).get("agent_name")
        agent_config = load_agent_config(agent_name)

        if not agent_config or not agent_config.hitl_keywords:
            return

        # 检查当前任务描述是否包含敏感关键词
        current_task = _get_current_task(state)
        if _matches_hitl_keywords(current_task, agent_config.hitl_keywords):
            # 使用 LangGraph 原生 interrupt()，替代 Laifu 的 sentinel 字符串
            from langgraph.types import interrupt
            approval = interrupt({
                "type": "hitl_approval",
                "agent": agent_name,
                "task": current_task,
                "message": f"域 Agent [{agent_name}] 将执行敏感操作，请确认或拒绝。",
            })
            if not approval.get("approved"):
                raise HITLRejectedError(f"操作被用户拒绝: {current_task}")
```

将 `HITLMiddleware` 注入到 `backend/src/agents/lead_agent/agent.py` 的 `_build_middlewares()` 中，位于 `SandboxMiddleware` 之后。

### Step 3.2：LangGraph 原生中断替代 Sentinel 字符串（2 天）

检查现有代码中是否有 sentinel 字符串残留（`__YIELD_TRIGGERED__`、`__ASK_HUMAN__` 等），全部替换为 LangGraph 原生机制：

```python
# 替换前（Laifu 风格）
return f"__YIELD_TRIGGERED__:{missing_information_desc}"

# 替换后（LangGraph 原生）
from langgraph.types import interrupt, Command
interrupt({"type": "yield", "reason": missing_information_desc})
# 或路由控制
return Command(goto="router")
```
