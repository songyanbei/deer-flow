请对你刚完成的开发工作进行自我审查。

## 审查清单

### 功能完整性
- [ ] 对照 checklist，每一项是否都已完成
- [ ] 边界情况是否处理（空值、超长输入、并发）
- [ ] 错误路径是否有合理的降级/提示

### 代码质量
- [ ] 没有遗留 TODO / FIXME / HACK 注释
- [ ] 没有调试代码（print、console.log）
- [ ] 没有硬编码的配置值（应使用环境变量或配置文件）
- [ ] 没有重复代码

### 安全性
- [ ] 所有用户输入都经过验证
- [ ] 没有敏感信息泄露（token、密码、密钥）
- [ ] 权限检查完善（tenant 隔离、角色校验）

### 测试
- [ ] 新增功能有对应的单元测试
- [ ] 测试全部通过
- [ ] 覆盖正常路径和异常路径

### 完成后
1. 总结你的改动（修改了哪些文件、新增了什么功能）
2. 列出可能的风险点
3. 创建 PR，标题简明，描述包含 checklist 完成情况
