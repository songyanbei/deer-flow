# 平台接入改造实施方案

更新时间：2026-03-30

## 1. 文档定位

本文档是 DeerFlow 多智能体框架接入外部智能体平台的完整实施方案，覆盖三大改造任务：

1. **OIDC 认证接入** — 对接平台认证中心，所有请求具备身份
2. **多租户隔离** — 数据按租户隔离，阻止跨租户访问
3. **智能体管理中心交互** — 子智能体 CRUD 同步 + 组合智能体运行时分组

三项任务有依赖关系，必须按顺序推进：认证 → 多租户 → 智能体交互。

## 3. 总体架构

### 3.1 目标架构

```
平台智能体管理中心 (Source of Truth)
│
├── 管理面：子智能体 CRUD / MCP 配置 / 技能管理
│   ↓
│   DeerFlow Gateway (:8001)
│   ├── OIDCAuthMiddleware (验 JWT)
│   ├── TenantContext 注入 (request.state.tenant_id / user_id)
│   ├── ThreadRegistry 归属校验
│   └── /api/agents, /api/mcp, /api/skills, ...
│
├── 运行面：对话执行
│   ↓
│   Nginx (:2026) → token 校验 → 透传 header
│   ↓
│   LangGraph Server (:2024)
│   ├── context.allowed_agents → 组合智能体分组过滤
│   ├── context.tenant_id → 贯穿 planner/router/executor
│   └── context.user_id → 线程归属
│
└── 数据面：租户隔离
    .deer-flow/
    ├── tenants/{tenant_id}/
    │   ├── memory.json
    │   ├── agents/{name}/config.yaml + SOUL.md
    │   └── governance_ledger.jsonl (或字段过滤)
    └── threads/{thread_id}/user-data/  (不变)
```

### 3.2 设计原则

1. **不动平台配置层** — config.yaml、MCP 连接池、引擎注册表对所有租户共用
2. **只隔离数据层** — 线程文件、记忆、治理记录、Agent 定义按 tenant 隔离
3. **认证在边界做** — Gateway/Nginx 层验 token，不改 LangGraph 内核
4. **分组关系存平台** — DeerFlow 不存储组合智能体定义，每次请求由平台声明 `allowed_agents`
5. **thread 归属是安全核心** — 建立 thread_id → tenant_id 映射，所有访问控制基于此

#### 4.1.2 新增 `src/gateway/middleware/auth.py`

```python
"""OIDC JWT verification middleware."""

import logging
from datetime import datetime, timezone

import httpx
from fastapi import Request
from jose import JWTError, jwt
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.responses import JSONResponse

logger = logging.getLogger(__name__)

# ── 配置（已从 Keycloak Discovery 端点确认）──────────────
OIDC_ISSUER = os.getenv("OIDC_ISSUER", "https://20.20.136.3:8443/realms/moss")
OIDC_JWKS_URI = os.getenv("OIDC_JWKS_URI", "https://20.20.136.3:8443/realms/moss/protocol/openid-connect/certs")
OIDC_AUDIENCE = os.getenv("OIDC_AUDIENCE", "moss-market")
OIDC_ALGORITHMS = ["RS256"]
# 内网自签证书，需要跳过 SSL 验证
OIDC_VERIFY_SSL = os.getenv("OIDC_VERIFY_SSL", "false").lower() == "true"

# 免认证路径
AUTH_EXEMPT_PATHS = {"/health", "/docs", "/redoc", "/openapi.json"}

# ── JWKS 缓存 ────────────────────────────────────────
_jwks_cache: dict | None = None
_jwks_fetched_at: float = 0
JWKS_CACHE_TTL = 3600  # 1 小时


async def _get_jwks() -> dict:
    """获取并缓存 JWKS 公钥集。"""
    global _jwks_cache, _jwks_fetched_at
    now = datetime.now(timezone.utc).timestamp()
    if _jwks_cache and (now - _jwks_fetched_at) < JWKS_CACHE_TTL:
        return _jwks_cache
    async with httpx.AsyncClient(verify=OIDC_VERIFY_SSL) as client:
        resp = await client.get(OIDC_JWKS_URI)
        resp.raise_for_status()
        _jwks_cache = resp.json()
        _jwks_fetched_at = now
    return _jwks_cache


def _find_signing_key(jwks: dict, token: str) -> dict:
    """从 JWKS 中匹配 token header 的 kid。"""
    unverified_header = jwt.get_unverified_header(token)
    kid = unverified_header.get("kid")
    for key in jwks.get("keys", []):
        if key.get("kid") == kid:
            return key
    raise JWTError(f"No matching key found for kid={kid}")


class OIDCAuthMiddleware(BaseHTTPMiddleware):
    """
    验证 Authorization: Bearer <jwt> 请求头。
    解出 tenant_id 和 user_id 注入 request.state。
    """

    async def dispatch(self, request: Request, call_next):
        # 免认证路径
        if request.url.path in AUTH_EXEMPT_PATHS:
            return await call_next(request)

        # 提取 token
        auth_header = request.headers.get("Authorization", "")
        if not auth_header.startswith("Bearer "):
            return JSONResponse(status_code=401, content={"detail": "Missing Bearer token"})
        token = auth_header.removeprefix("Bearer ").strip()

        try:
            jwks = await _get_jwks()
            signing_key = _find_signing_key(jwks, token)
            payload = jwt.decode(
                token,
                signing_key,
                algorithms=OIDC_ALGORITHMS,
                audience=OIDC_AUDIENCE,
                issuer=OIDC_ISSUER,
            )
        except JWTError as e:
            logger.warning("JWT verification failed: %s", e)
            return JSONResponse(status_code=401, content={"detail": f"Invalid token: {e}"})
        except httpx.HTTPError as e:
            logger.error("Failed to fetch JWKS: %s", e)
            return JSONResponse(status_code=503, content={"detail": "Auth service unavailable"})

        # 注入租户上下文
        # tenant 字段名通过环境变量配置，待平台确认后填入
        _tenant_claim = os.getenv("OIDC_TENANT_CLAIM", "organization")
        request.state.tenant_id = payload.get(_tenant_claim) or "default"
        request.state.user_id = payload.get("sub", "anonymous")
        request.state.claims = payload  # 保留完整 claims 供后续使用

        return await call_next(request)
```

#### 4.1.4 Nginx 层保护 LangGraph Server

LangGraph Server (port 2024) 没有内置认证。在 Nginx 层拦截无 token 请求：

```nginx
# docker/nginx/nginx.local.conf
location /api/langgraph/ {
    # 要求 Authorization header 存在
    if ($http_authorization = '') {
        return 401 '{"detail": "Missing Authorization header"}';
    }
    # 透传给 LangGraph Server
    proxy_pass http://langgraph:2024/;
    proxy_set_header X-Tenant-ID $http_x_tenant_id;
    proxy_set_header X-User-ID $http_x_user_id;
}
```

> 注意：Nginx 做的是**粗粒度拦截**（header 必须存在），细粒度 JWT 验签在 Gateway 做。
> 如果 LangGraph 请求也需要严格验签，后续可在 Nginx 加 `auth_request` 子请求。

#### 4.2.1 Thread 归属注册

**新增 `src/gateway/thread_registry.py`**

```python
"""Thread → Tenant 归属映射。"""

import json
import threading
from pathlib import Path

from src.config.paths import get_paths


class ThreadRegistry:
    """
    维护 thread_id → tenant_id 映射。
    存储为 JSON 文件：{base_dir}/thread_registry.json
    """

    def __init__(self):
        self._lock = threading.Lock()

    @property
    def _file(self) -> Path:
        return get_paths().base_dir / "thread_registry.json"

    def _load(self) -> dict[str, str]:
        if self._file.exists():
            return json.loads(self._file.read_text(encoding="utf-8"))
        return {}

    def _save(self, data: dict[str, str]) -> None:
        self._file.parent.mkdir(parents=True, exist_ok=True)
        self._file.write_text(json.dumps(data, ensure_ascii=False, indent=2), encoding="utf-8")

    def register(self, thread_id: str, tenant_id: str) -> None:
        with self._lock:
            data = self._load()
            data[thread_id] = tenant_id
            self._save(data)

    def get_tenant(self, thread_id: str) -> str | None:
        return self._load().get(thread_id)

    def check_access(self, thread_id: str, tenant_id: str) -> bool:
        owner = self.get_tenant(thread_id)
        return owner is None or owner == tenant_id  # 未注册的线程放行（兼容存量）

    def list_threads(self, tenant_id: str) -> list[str]:
        return [tid for tid, tid_tenant in self._load().items() if tid_tenant == tenant_id]


thread_registry = ThreadRegistry()
```

```python
"""FastAPI 依赖注入 — 租户校验。"""

from fastapi import HTTPException, Request


def get_tenant_id(request: Request) -> str:
    """从 request.state 获取 tenant_id（由 OIDCAuthMiddleware 注入）。"""
    tenant_id = getattr(request.state, "tenant_id", None)
    if not tenant_id:
        raise HTTPException(status_code=401, detail="Tenant context missing")
    return tenant_id


def get_user_id(request: Request) -> str:
    """从 request.state 获取 user_id。"""
    return getattr(request.state, "user_id", "anonymous")
```

#### 4.3.2 批量同步接口

**新增 `POST /api/agents/sync`**：

```python
class AgentSyncRequest(BaseModel):
    """批量同步子智能体。"""
    agents: list[AgentCreateRequest]
    mode: Literal["upsert", "replace"] = "upsert"
    # upsert: 存在则更新，不存在则创建
    # replace: 删除不在列表中的 agent（需确认）

class AgentSyncResponse(BaseModel):
    created: list[str]
    updated: list[str]
    deleted: list[str]
    errors: list[dict]

@router.post("/api/agents/sync", response_model=AgentSyncResponse)
async def sync_agents(request: Request, body: AgentSyncRequest):
    tenant_id = get_tenant_id(request)
    created, updated, deleted, errors = [], [], [], []

    existing = {a.name for a in list_domain_agents(tenant_id=tenant_id)}
    incoming = {a.name for a in body.agents}

    for agent_req in body.agents:
        try:
            if agent_req.name in existing:
                # 更新
                await update_agent(agent_req.name, AgentUpdateRequest(...))
                updated.append(agent_req.name)
            else:
                # 创建
                await create_agent_endpoint(agent_req)
                created.append(agent_req.name)
        except Exception as e:
            errors.append({"name": agent_req.name, "error": str(e)})

    if body.mode == "replace":
        for name in existing - incoming:
            await delete_agent(name)
            deleted.append(name)

    return AgentSyncResponse(created=created, updated=updated, deleted=deleted, errors=errors)
```
